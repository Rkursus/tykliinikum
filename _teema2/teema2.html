Rakendustarkvara: R

1 Vrtuste tbid.

Nagu eelmises teemas eldud, siis vektori vrtused peavad olema kik hte tpi. R-is on vrtuste phitbid:

  • int / integer tisarvud (ka negatiivsed)
  • numeric reaalarvud
  • cplx / complex kompleksarvud
  • char / character sned (themrgid ja muud tekstilised smbolid, sisestamisel kasutada jutumrke)
  • logical tevrtused (ainult kaks vrtust: TRUE vi FALSE)

ht tpi vrtust saab vahel teisendada teist tpi vrtuseks vastava as.<tbi_nimi> ksuga (as.integer, as.numeric, as.character jne). Kontrollimaks mingi vrtuse tpi saab kasutada vastavat ksku is.<tbi_nimi> (is.integer, is.numeric, is.character jne).

Eelmises teema juures moodustasime mned vektorid. Tekitame need uuesti

kaalud <- c(7, 3.5, 0.4, 2, 3.2, 20.2)
liik <- c("koer", "kass", "rott", "kass", "kass", "koer")
vanused <- c(7, NA, 3, 53, 53, 95)

ja kontrollime vrtuste tpe:

is.integer(kaalud)
## [1] FALSE
is.numeric(kaalud)
## [1] TRUE
class(liik)
## [1] "character"
class(vanused)
## [1] "numeric"

Eraldi ksk on ka puuduvate vrtuste esinemise kontrolliks:

is.na(vanused)
## [1] FALSE  TRUE FALSE FALSE FALSE FALSE

1.0.1 lesanded

  2. Miks annavad ksud is.integer(1:4) ja is.integer(c(1, 2, 3)) erineva tulemuse?
  3. Rakenda funktsiooni is.na(.) vektorile z <- c("a", "NA", NA, 0). Kas tulemus on oodatav?

1.1 Tevrtused ja tevrtusvektorid

Kui ksuga mean kasutada argumenti na.rm, siis selle argumendi vrtus vib olla TRUE vi FALSE. Tegemist on tevrtustega (ingl. k. boolean). Kuna tevrtustega saab teha loogilisi tehteid (ehk neid kombineerida), siis on neist kasu ka mujal kui na.rm argumendi vrtustamisel.

Loogilisi tehteid on kolm: korrutamine (&), liitmine (|) ja eitus (!).

tehe tulemus
TRUE & TRUE TRUE
TRUE & FALSE FALSE
FALSE & TRUE FALSE
FALSE & FALSE FALSE
TRUE | TRUE TRUE
TRUE | FALSE TRUE
FALSE | TRUE TRUE
FALSE | FALSE FALSE
!TRUE FALSE
!FALSE TRUE

Tevrtus on tulemuseks siis, kui vrdleme kahte objekti/vrtust omavahel.

vrdlus smbol nide
vrdumine == 2 == 3 ; "a" == "A"
mittevrdumine != 2 != 3 ; "a" != "A"
viksem kui < 2 < 3 ; "a" < "A"
viksem vi vrdne <= 3 <= 2 ; "a" <= "A"
suurem kui > 3 > 2 ; "a" > "A"
suurem vi vrdne >= 3 >= 2 ; "a" >= "A"

Nagu ennist mainitud, siis R-is tehakse tehteid vektoritega elementhaaval. Seeprast ka siis, kui vtame mingi arvulise vektori ja vrdleme seda mingi arvuga, siis vrreldakse igat elementi eraldi ja tulemuseks on tevrtustest koosnev vektor, milles on sama palju elemente kui oli elemendiviisilisi vrdluseid. Sama kehtib ka juhul, kui vrreldakse muud tpi vrtuseid (niteks tekstilisi vrtuseid).

kaalud > 10
vanused == 53
liik == "kass"
vanused == kaalud
liik == "kass" | liik == "koer"

Mnikord on soov kontrollida, kas mingi vrtus leidub etteantud hulgas. Siis on sobilik kasutada operaatorit %in% :

1:4 %in% c(2, 5)
## [1] FALSE  TRUE FALSE FALSE
c(2, 5) %in% 1:4
## [1]  TRUE FALSE

Tevrtuste puhul on huvitav see, et kui nendega tavalisi arve korrutada vi liita vms, siis konverteeritakse tevrtused arvudeks: TRUE muutub arvuks 1 ja FALSE muutub arvuks 0.

is.na(vanused) * 1
## [1] 0 1 0 0 0 0
sum(is.na(vanused))  # mitmel vaatlusalusel on vanus puudu
## [1] 1

Konverteerimine toimib automaatselt. Kui on endal soov tevrtuseid arvuliseks muuta, saab rakendada ksku as.numeric(.) tevrtustega vektorile. Saab ka vastupidi: kui on soov arvusid tevrtusteks muuta, saab seda teha ksuga as.logical(.). Proovi!

  2. Selgita vlja, millised arvud konverteeritakse vrtuseks TRUE ja millised vrtuseks FALSE, kui kasutada ksku as.logical.
  3. Mis on loogiliste tehete tulemus, kui ks tehtes osalev vrtus on NA?

2 Andmestik, andmete import

Tavaliselt koosnevad andmestikud ridadest ja veergudest (tulpadest), kus iga rida vastab mingile mtmisobjektile ja iga veerg vastab mingile mdetud omadusele (tunnusele).

Erinevad statistikaprogrammid kasutavad andmestike silitamiseks eri failiformaate. Et andmestikke hest programmist teise saada, on heks vimalikuks lahenduseks andmetabel salvestada vahepeal tekstiformaadis failiks, niteks .txt vi .csv tpi failiks ja importida tekstifail.

MS Exceli faile ning statistikatarkvarade spetsiifilisi andmeformaate (nt Stata .dta, SPSS-i, .sav) ei saa R-i importida baaspaketi kskude abil. Selliste failide impordiks on vaja kasutada mnda R-i lisapaketti.

Nd vaatame nii tekstifailis andmete kui ka mne teise statistikaprogrammi andmefailide importimist R-i.

2.1 Andmete sisselugemine ja faili kirjutamine (tekstifail)

Kige olulisem ksk tekstikujul andmestike sisselugemiseks on read.table(). Sellel on mitmeid argumente, mille abil saab tpsustada erinevaid asjaolusid, mis antud andmestikku puutuvad:

file - faili asukoht ja nimi koos faililaiendiga (peab olema jutumrkides)
header - kas faili esimeses reas on veerunimed? (TRUE = jah, FALSE = ei)
sep - millise smboliga on veerud eraldatud? (nt "\t" - tabulaatori smboliga)
dec - kuidas on mrgitud kmnendmurru eraldaja? (nt "." vi ",")
quote - millega on tekstilised vrtused mbritsetud? (nt "\"" thendab, et kahekordsete jutumrkidega)
na.strings - kuidas on failis puuduvad vrtused thistatud?
fileEncoding - mis thekodeeringut kasutatakse (Windowsis sageli "latin9", Macis/Linuxis sageli "utf8")

Faili asukoht vib olla kvakettal vi vrgus. Kui ksu setwd abil on mratud, milline on kimasoleva tsessiooni tkataloog, ja andmefail on selles kataloogis, siis piisab ksule read.table ainult faili nime etteandmisest (tispikka asukohta ei pea andma). Windowsis on kombeks kaustastruktuuri thistamiseks kasutada kurakaldkriipsu (tagurpidi kaldkriipsu) \, aga R-is on sellisel kaldkriipsul eriline thendus sellega mrgitakse, et jrgneb erismbol (nt \t on tabulaatori smbol). Seeprast tuleb kataloogitee mrkimisel kasutada kahekordseid kurakaldkriipse, niteks:

setwd("C:\\Users\\mina\\Rkursus\\")

Teine vimalus on kasutada tavalist kaldkriipsu, nagu MacOS-is ja Linuxites:

setwd("C:/Users/mina/Rkursus/")

Andmete sisselugemisel tuleks anda andmetabelile nimi (salvestada see mingi objektina), vastasel juhul andmestik trkitakse lihtsalt ekraanile ja sellega enam midagi muud teha ei saa:

naide1 <- read.table("https://github.com/Rkursus/tykliinikum/raw/master/data/esimene.txt", 
                     header = T, sep = "\t", dec = ",")

Tkeskkonnas olevaid andmetabeleid saab faili kirjutada ksuga write.table(.), andes ksule ette salvestatava andmestiku nime ning loodava faili nime jutumrkides (vajaduse korral koos kataloogiteega):

write.table(nide1, "failinimi.txt", sep = "\t")

2.2 Esmase levaate saamine andmetabelist

Ksuga read.table(.) sisse loetud andmestik on erilist tpi, data.frame-tpi objekt. Andmestikust saab kiire levaate jrgmiste kskudega:

nrow(andmed) - mitu rida on andmestikus
ncol(andmed) - mitu veergu on andmestikus
dim(andmed) - mitu rida ja mitu veergu on andmestikus
str(andmed) - andmestiku struktuur: mis tpi iga tunnus on ja esimesed vrtused
summary(andmed) - lhike kirjeldav statistika iga tunnuse kohta
names(andmed) - veergude nimed
head(andmed) - andmestiku mned esimesed read

Ksu summary(.) puhul on nha, et mne tunnuse puhul arvutatakse keskmine, miinimum, maksimum jne, aga teise tunnuse puhul esitatakse sagedused. See, millist kirjeldusviisi kasutatakse, tuleneb tunnuse tbist. Kui ksuga str(.) vaadata, mis on tunnuste tbid selles andmestikus, on nha kahte tpi tunnuseid: int ja Factor. On nha, et summary(.) ksk arvutab int-tpi tunnustele keskmisi jne, Factor-tpi tunnustele aga sagedusi. Faktortpi tunnus sarnaneb snele (character), kuid heks erinevuseks on niteks fikseeritud vrtuste hulk.

2.3 Lisapakettide kasutamine

ks R-i populaarseks muutumise phjuseid on rikkalik lisapakkettide olemasolu. Tenoliselt leidub iga praktilise statistika-alase (ja ka mne muu valdkonna) probleemi jaoks omaette pakett (package). Niteks paketid foreign, haven, readxl sisaldavad funktsioone, mis aitavad erinevates formaatides andmestikke hlpsamini R-i sisse lugeda. Andmete impordiks sobilikke lisapakette on lisaks nimetatutele veel. Siinkohal vaatame aga kahe paketi haven ja readxl vimalusi.

Lisapaketid ei ole tavaliselt R-iga kaasas, vaid need tuleb installeerida. Kui paigaldatav pakett vajab omakorda mingeid muid pakette, siis installeeritakse ka need. Mnikord vib R pakette installeerides ksida millisest serverist soovib kasutaja neid alla laadida. Kuna R-i taga on suur vrgustik, siis allikaid leidub le maailma ja tasub valida Eestile lhim allikas. Jrgmistel kordadel sama tsessiooni jooksul pakette paigaldades seda enam ei ksita.

Installeerime lisapaketid readxl ja haven

install.packages("readxl")
install.packages("haven")
# NB! Arvutiklassi arvuti korral vib olla vajalik salvestuskoha ettemramine
# install.packages("readxl", lib = "C:/kataloog-paigaldamiseks")

Kui pakett on paigaldatud, siis jrgmisel tsessioonil seda enam uuesti paigaldama ei pea. Kll aga tuleb arvutisse paigaldatud paketid igal uuel R-i sessioonil sisse laadida ksuga library (vi require), mille argumendiks on laaditava paketi nimi:

library(readxl)
# Kui paigaldamisel oli ksitsi mratud salvestuskoht:
# library(readxl, lib.loc = "C:/kataloog-installimiseks")

RStudio on tulnud mitmes kohas appi ja seda ka andmete importimisel. Nimelt on paremal aknas Environment vahelehel nupp Import Dataset. Valides seal sobiva programmi ja seadistades nii, et impordi eelvaade on selline nagu peab. Siis piisab kui vajutada lihtsalt Import nuppu.

2.4.1 Nide Exceli faili impordist paketi readxl ja konsooli abil

Lisapaketi readxl ksu read_excel(.) abil saab importida MS Exceli failide (nii .xlsx kui xls) tlehti. Salvesta aadressilt https://github.com/Rkursus/tykliinikum/tree/master/data/ oma tkataloogi fail tudengite-arv.xlsx, mis sisaldab infot T tudengite arvude kohta aastate likes. Ava esmalt fail MS Exceli abil ning vaata millised tlehed failis on. Seejrel vaata ka R-i abil millised on tlehtede nimed ja impordi teine tleht, millel on andmed kunagise avatud likooli pilaste arvude kohta.

list.files() # vaata, mis nimega failid on tkaustas
excel_sheets("tudengite-arv.xlsx") # tlehtede nimed MS Exceli failis
AY <- read_excel("tudengite-arv.xlsx", sheet = "avatud ylikool") # importimine
# AY <- read_excel("tudengite-arv.xlsx", sheet = 2)  # sama
str(AY)  # vaata tulemust

MS Exceli failis ei pruugi andmed alata alati esimesest reast, niteks tlehel tabel on esimeses reas tabeli nimi ning teine rida on thi, andmetabel algab kolmandalt realt. Selle, et esimesed kaks rida tuleks andmete impordil vahele jtta, saab ksule read_excel(.) ette anda argumendi skip abil:

tabel <- read_excel("tudengite-arv.xlsx", sheet = "tabel", skip = 2)
str(tabel)

Kui andmete ttlemine on lpetatud, siis saab R-i andmestiku haven paketi abil ka eksportida vaadeldud programmide andmefailiks ksuga write_<...>(.), kus <...> tuleks siis asendada sobiliku programminimega, nt. write_sas() vi write_sav.

Samas ei pruugi need ksud siiski olla 100% tkindlad olla. Niteks vib juhtuda, et write_sas() ksu tulemust uuem SAS omaks ei tunnista vi osa informatsioonist on puudu (nt. tunnuste kodeeringud SPSS *.sav failist)). Sellisel on esimesed sammud:

  • uuendage pakett haven;
  • eksportida R-st fail ja SAS-i/SPSS-i importimisel elda, et tegemist on vanema versiooniga;
  • kaduma linud informatsioon R-st eraldi eksportida ja liita SAS/SPSS-s.
